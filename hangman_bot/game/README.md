### Hangman: Game

В этой подзадаче необходимо реализовать класс игровой логики для одной партии в «Виселицу».

Класс не должен зависеть от Telegram API, файловой системы и других внешних компонентов. Его задача — хранить состояние игры и обновлять его после каждой попытки угадывания.

##### Идея

Объект `HangmanGame` представляет одну партию и хранит:

* загаданное слово;
* текущую маску угадывания (неотгаданные буквы заменены специальным символом);
* количество неверных попыток.

##### Ожидаемый интерфейс

Рекомендуемый набор методов:

* `bool Guess(char letter)`
  * принимает букву;
  * открывает все вхождения буквы в слове;
  * увеличивает счетчик ошибок, если буквы нет;
  * возвращает `true`, если буква была в слове, иначе `false`.

* `bool IsWin() const`
  * возвращает `true`, если текущее состояние совпало с загаданным словом.

* `bool IsLose() const`
  * возвращает `true`, если число ошибок достигло лимита (`kMaxWrongGuesses`).

* `std::string_view GetGuess() const`
  * возвращает текущее отображение слова (маску).

* `std::string_view GetWord() const`
  * возвращает исходное загаданное слово.

* `int GetWrongGuesses() const`
  * возвращает количество неверных попыток.

##### Детали

1. Повторяющиеся буквы должны открываться сразу во всех позициях.
2. Состояние должно обновляться за O(n) по длине слова на одну попытку (или лучше).
3. Класс должен оставаться простым value-type объектом, пригодным для сериализации/десериализации.

##### Замечания

1. В качестве решения ожидается реализация в `hangman.hpp` (и при необходимости `hangman.cpp`).
2. Логику работы с пользователем/чатом в этот класс добавлять не нужно.

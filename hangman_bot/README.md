### Hangman Bot

В этой задаче необходимо реализовать Telegram-бота для игры в "Виселицу".

Проект разбит на 3 подзадачи:

* `game` — логика одной игровой сессии (слово, текущее состояние угадывания, число ошибок);
* `word_generator` — загрузка слов из файла и выдача случайного слова;
* `bot` — интеграция с Telegram Bot API через библиотеку TgBot, обработка команд и сообщений, хранение состояния игр пользователей.

##### Что нужно реализовать

1. Класс игры (`game`):
   * хранение загаданного слова;
   * обработка попытки угадывания буквы;
   * проверка состояний победы/поражения;
   * выдача текущей маски слова.

2. Генератор слов (`word_generator`):
   * загрузка слов из текстового файла;
   * фильтрация невалидных слов;
   * нормализация регистра;
   * равномерный случайный выбор слова.

3. Класс бота (`bot`):
   * обработка команд `/start`, `/help`, `/newgame`;
   * обработка пользовательских сообщений с буквами;
   * выдача ответов в чат;
   * сохранение/восстановление состояния игр при перезапуске.

##### Зависимости (Ubuntu)

Для сборки потребуются:

* компилятор C++20 (`g++`);
* `cmake`;
* `libssl-dev`;
* `zlib1g-dev`;
* `libcurl4-openssl-dev`;
* библиотека `TgBot`.

Пример установки:

```bash
sudo apt update
sudo apt install -y g++ cmake libssl-dev zlib1g-dev libcurl4-openssl-dev
```

Библиотека `TgBot` должна быть установлена в систему (например, как `libTgBot.a`/`libTgBot.so` в стандартных путях линковщика или в `/usr/local/lib`).
Инструкция: https://github.com/reo7sp/tgbot-cpp

##### Сборка проекта

Из папки `build`:

```bash
cmake ..
make bot
```

##### Запуск бота

Перед запуском бота, его необходимо создать. Делается это через `@BotFather` в Telegram.

Создайте бота по инструкции и сохраните токен доступа к нему в текстовом файле (например, `token.txt`).

ВАЖНО: не публикуйте токен в открытых репозиториях и не делитесь им с посторонними, так как он позволяет управлять вашим ботом.

Исполняемый файл принимает аргументы:

```text
bot <telegram_token_path> <words_path> <state_path> [seed]
```

Где:

* `telegram_token_path` — путь к файлу с токеном (токен в первой строке файла);
* `words_path` — путь к файлу слов (по одному слову на строку);
* `state_path` — путь к файлу состояния игр (создается/обновляется автоматически);
* `seed` — необязательное зерно генератора случайных чисел для воспроизводимости.

Пример:

```bash
./build/bot ./hangman_bot/token.txt ./hangman_bot/word_generator/words.txt ./hangman_bot/state.txt
```

##### Замечания

1. Рекомендуется реализовывать подзадачи последовательно: `game` -> `word_generator` -> `bot`.
2. Хранение состояния игр можно реализовать через map в памяти с периодическим/событийным сохранением на диск.
3. Для корректной отладки удобнее использовать фиксированный `seed`.

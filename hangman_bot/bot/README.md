### Hangman: Bot

В этой подзадаче необходимо реализовать класс `HangmanTgBot`, который связывает игровую логику и генератор слов с Telegram.

Класс должен:

* принимать входящие обновления из Telegram;
* обрабатывать команды и сообщения пользователя;
* поддерживать отдельные игровые сессии для разных чатов;
* сохранять состояние игр, чтобы они восстанавливались после перезапуска процесса.

##### Архитектура

Минимальная рекомендуемая схема:

* `HangmanGame` — доменная логика партии;
* `WordGenerator` — источник случайных слов;
* `HangmanTgBot` — orchestration слой:
  * регистрация обработчиков Telegram;
  * маршрутизация событий;
  * хранение `chat_id -> game` (например, `std::unordered_map<int64_t, HangmanGame>`);
  * загрузка/сохранение состояния.

##### Что нужно реализовать в боте

1. Команда `/start`
   * краткая приветственная инструкция.

2. Команда `/help`
   * подсказка по доступным действиям.

3. Команда `/newgame`
   * создание новой игры для текущего чата;
   * выбор слова через `WordGenerator`;
   * сохранение состояния.

4. Текстовые сообщения
   * если это не команда и длина сообщения == 1, трактовать как попытку угадать букву;
   * обновлять `HangmanGame`;
   * отправлять ответ пользователю;
   * сохранять состояние.

##### Telegram API (TgBot): необходимые классы и методы

Ниже перечислены инструменты, которые используются в эталонной реализации.

**1. `TgBot::Bot`**

Главный объект бота. Создается из токена.

Типичное использование:

```cpp
TgBot::Bot bot(token);
```

Через этот объект вы получаете:

* API-клиент (`getApi()`);
* диспетчер событий (`getEvents()`).

**2. `TgBot::EventBroadcaster` через `bot.getEvents()`**

Набор регистраций обработчиков входящих обновлений.

Ключевые методы:

* `onCommand("start", handler)` — обработчик конкретной slash-команды;
* `onUnknownCommand(handler)` — обработчик неизвестной команды;
* `onAnyMessage(handler)` — обработчик любого входящего сообщения.

Что важно:

* `handler` - это функция, принимающая указатель на принятое сообщение (`TgBot::Message::Ptr`);
* из `message` можно получить `message->chat->id` (идентификатор чата) и `message->text` (текст сообщения);
* исключения из handler лучше перехватывать локально, чтобы не срывать long polling цикл.

**3. `TgBot::Api` через `bot.getApi()`**

Клиент для вызовов методов Telegram Bot API.

Нужные методы:

* `sendMessage(chatId, text)`
  * отправка текстового сообщения в конкретный чат;
  * используется для всех ответов пользователю.

* `deleteWebhook()`
  * отключает webhook-режим;
  * обязателен перед long polling, чтобы Telegram не считал, что вы работаете через webhook.

* `getMe()`
  * возвращает информацию о текущем боте;
  * удобно для логов и проверки, что токен валидный.

**4. `TgBot::TgLongPoll`**

Класс long polling цикла, который забирает обновления и прокидывает их в зарегистрированные обработчики.

Типичное использование:

```cpp
auto long_poll = TgBot::TgLongPoll(bot);
while (true) {
  long_poll.start();
}
```

Рекомендации:

* окружить `start()` обработкой исключений;
* при ошибках логировать и продолжать цикл;
* не допускать выброса исключений из callback наружу.

**5. `TgBot::Message::Ptr`**

Умный указатель на входящее сообщение.

Используемые поля:

* `message->text` — текст сообщения;
* `message->chat->id` — идентификатор чата (ключ сессии).

**6. `TgBot::TgException`**

Исключения библиотеки TgBot. Обычно ловятся вокруг long polling цикла и API-вызовов.

##### Сохранение состояния игр

Базовый простой вариант:

* в памяти: `unordered_map<ChatId, HangmanGame>`;
* на диске: snapshot-файл с сериализованными состояниями;
* при старте: загрузить snapshot;
* при изменении игры: обновить map и сохранить snapshot.

Практические рекомендации:

1. Писать состояние через временный файл и атомарный `rename`.
2. Ошибки сохранения логировать, но не останавливать polling.
3. Валидацию строк при загрузке делать максимально строгой (битые записи пропускать).

##### Входные данные процесса

Рекомендуемый запуск:

```text
bot <telegram_token_path> <words_path> <state_path> [seed]
```

Где:

* `telegram_token_path` — файл с токеном в первой строке;
* `words_path` — словарь для `WordGenerator`;
* `state_path` — файл состояния игр;
* `seed` — необязательный параметр воспроизводимости.

##### Замечания

1. Реализация должна быть устойчивой к ошибкам сети/диска.
2. Бот не должен терять уже созданные игры при перезапуске.
3. Логику игры и генерацию слов не дублировать в `bot`, а вызывать соответствующие классы.
4. Более подробную инструкцию по работе с библиотекой TgBot и примеры использования ищите в репозитории: https://github.com/reo7sp/tgbot-cpp

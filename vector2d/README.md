# Моделирование физических процессов.

## Часть 2. Создание векторной математики для двумерных величин.

*В физическом моделировании нам часто приходится работать с векторными величинами, такими как скорость, ускорение или сила, имеющими направление и величину. В двумерном случае такие величины можно представить вектором из двух компонент (x и y). Правильная реализация операций над векторами позволит выполнять вычисления более наглядно и при этом сократить количество ошибок.*

*Например, для изменения скорости тела под действием силы (которую можно представить вектором), достаточно сложить векторы скоростей или умножить их на скаляр (масштабирование). Реализация набора операций над векторами станет фундаментом для последующего моделирования движения тел под действием различных сил.*

### Требуется реализовать

1. **Шаблон структуры `Vector2D`**

   - Определить шаблонную структуру `Vector2D<T>`, которая хранит две компоненты: `x` и `y` типа `T`.  
   - Предполагается использовать `Vector2D<T>` для хранения векторов любой размерности величины `T`, в том числе физических величин (как, например, `Length`, `Speed` и т.д.).

2. **Операции над векторами**

   - **Сложение и вычитание двух векторов одинакового типа**  
     Реализовать операции `+` и `-` для двух `Vector2D<T>` с одинаковым типом `T`. Они должны поэлементно складывать или вычитать соответствующие компоненты.
     
     Также реализовать операции `+=` и `-=` для модификации вектора с учётом другого вектора.

   - **Унарный плюс и минус**  
     Реализовать операции унарных `+v` и `-v`, где `v` — вектор, чтобы получить вектор с тем же направлением и знаком (`+v`) или с изменением знака каждого компонента (`-v`).

   - **Умножение и деление на скаляр**  
     Реализовать операции умножения вектора на произвольный скаляр `c`: `v * c` и `c * v`. Они должны умножать каждую компоненту `v.x` и `v.y` на скаляр `c`.
     
     Аналогично реализовать деление вектора на скаляр `v / c`.  
     Добавить операции с присваиванием `*=`, `/=` для умножения и деления вектора на скаляр.

   - **Скалярное произведение**  
     Реализовать функцию `Dot(lhs, rhs)`, возвращающую скалярное произведение двух векторов `Vector2D<T>` и `Vector2D<U>` как `lhs.x * rhs.x + lhs.y * rhs.y`.

   - **Важно**
     Обратите внимание, что умножение (деление) вектора `Vector2D<T>` на скаляр, не обязательно возвращает `Vector2D<T>`. Например, умножение вектора целых чисел на дробное число возвращает вектор дробных чисел, а умножение вектора скорости на время дает вектор перемещения (длины). Аналогично, скалярное произведение векторов разных размерностей не обязательно приводит к типу `T` или `U`. В этих ситуациях, когда непонятен тип возвращаемого значения, вывод типа можно предоставить компилятору, указав `auto` вместо возвращаемого типа.

3. **Операции сравнения**

   - Реализовать операции равенства (`==`) и неравенства (`!=`) для векторов, чтобы можно было проверить, равны ли соответствующие компоненты.

### Дополнительные пояснения

- Код должен быть максимально универсален. Если `T` --- тип величины (например, `Length`), то операции сложения, вычитания или умножения на скаляр должны корректно работать с этим типом.
- При использовании этого векторного типа в физической модели можно будет удобно складывать скорости, ускорения, силы или любые другие векторные величины, не заботясь о ручном поэлементном вычислении.

### Примеры использования

**Пример 1: Сложение векторов**

```cpp
Vector2D<double> v1{3.0, 4.0};
Vector2D<double> v2{1.0, 2.0};

auto v3 = v1 + v2; // v3 = {4.0, 6.0}
```

**Пример 2: Умножение на скаляр**

```cpp
Vector2D<double> v{1.0, -2.0};
auto v2 = v * 2.0;   // v2 = {2.0, -4.0}
auto v3 = 0.5 * v;   // v3 = {0.5, -1.0}
v *= -1;             // v = {-1.0, 2.0}
```

**Пример 3: Скалярное произведение**

```cpp
Vector2D<double> a{1.0, 2.0};
Vector2D<double> b{3.0, 4.0};
double dot = Dot(a, b); // dot = 1*3 + 2*4 = 11
```

---

Таким образом, после реализации данного набора операций над `Vector2D<T>` мы получим удобный инструмент для работы с векторами любых физических величин, который станет фундаментом для дальнейшей симуляции механического движения.

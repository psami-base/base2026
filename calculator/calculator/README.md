### Calculator

Поздравляем, вы добрались до «Босса». В этом задании вы научитесь разбирать арифметические выражения и реализуете
практически полноценный калькулятор.

Основные идеи разбора арифметических выражений во всех подробностях представлены
[здесь](https://sites.google.com/site/slipstak2/arithm_article.pdf?attredirects=0&d=1).

*TL;DR:* Арифметическое выражение состоит из суммы одного или нескольких слагаемых. Слагаемое – это произведение (или
деление, или остаток от деления) нескольких множителей, а множитель – это либо константа, либо выражение, заключённое в
скобки. Таким образом, необходимо уметь выделять из исходного текста каждую из этих трёх сущностей ~~в виде гномика~~.

#### Парсинг множителя

Множитель определяется двумя простыми случаями:
- Если текущий токен – константа, то множитель равен этой константе.
- Если текущий токен – открывающая скобка, то множитель равен выражению, находящемуся внутри скобок. Для этого
  выполняется рекурсивный вызов парсера для разбора выражения внутри скобок.

#### Парсинг слагаемого

Парсинг слагаемого начинается с разбора одного множителя. После этого:
- Если следующий токен – операция `*`, `/` или `%`, парсится следующий множитель, а текущее значение слагаемого
  объединяется с новым посредством создания узла умножения, деления или остатка.
- Этот процесс повторяется до тех пор, пока встречаются операции множителей.

#### Парсинг выражения

Парсинг выражения начинается с разбора слагаемого. Затем:
- Если следующий токен – бинарная операция `+` или `-`, парсится следующее слагаемое, и текущее выражение комбинируется
  с новым посредством создания узла суммы или разности.
- Этот процесс повторяется до исчерпания токенов или до закрывающей скобки.

*Пример.* Рассмотрим выражение:
```
(2 + 3) * 4
```
Алгоритм разбора:

1. **Парсинг выражения.**  
   Начинается с парсинга слагаемого.
2. **Парсинг слагаемого.**  
   Вызывает парсинг множителя.
3. **Парсинг множителя.**  
   Видит открывающую скобку, поэтому рекурсивно парсится выражение внутри скобок.
4. **Внутреннее выражение.**  
   - Парсинг слагаемого внутри скобок начинает с разбора множителя, который видит константу `2` -> возвращается
     `Constant(2)`.
   - Затем обнаруживается операция `+`, поэтому парсится следующее слагаемое, получая `Constant(3)`.
   - Создаётся узел суммы: `Sum(Constant(2), Constant(3))`.
5. **Возврат из скобок.**  
   Полученное выражение `Sum(Constant(2), Constant(3))` становится множителем.
6. **Продолжение внешнего слагаемого.**  
   После множителя обнаруживается операция `*`, поэтому парсится следующий множитель:
   - Парсинг множителя видит константу `4` -> возвращается `Constant(4)`.
   - Создаётся узел умножения: `Multiply(Sum(Constant(2), Constant(3)), Constant(4))`.
7. **Завершение.**  
   Все токены разобраны, итоговое выражение возвращается, и вызов `Calculate()` вычисляет результат.

Полученное дерево выглядит так:

```
    *
   / \
  +   4
 / \
2   3
```

#### Детали реализации

Решение состоит из двух частей — токенизация (понадобится токенизатор из части 1) и построение дерева разбора (парсинг).

Используйте классы выражений из задачи `PolishNotation` (`expressions.hpp`). В этом задании вам понадобятся только операции
+, -, *, /, % и константы. Тем не менее подумайте, как поддержать возможность использования других операций
(например, функций).

В файлах `parser.hpp` и `parser.cpp` объявите и определите следующие функции:
- `ParseExpression`
- `ParseAddendum`
- `ParseMultiplier`

Каждая функция должна возвращать умный указатель (`std::unique_ptr<IExpression>`) и принимать аргументы:
- `const std::vector<Token>& tokens` — список токенов,
- `size_t& pos` — текущая позиция в массиве токенов.
 
В файлах `calculator.hpp` и `calculator.cpp` реализуйте функцию:
```cpp
int CalculateExpression(std::string_view input);
```
которая:
1. Токенизирует входную строку,
2. Строит дерево разбора,
3. Вычисляет и возвращает результат выражения.

Программа должна выбрасывать исключения:
- `UnknownSymbolError` — если токенизация обнаружила неизвестный токен,
- `WrongExpressionError` — если при разборе или вычислении выражения произошла ошибка.

Гарантируется, что все промежуточные результаты укладываются в тип `int`.
